# RFC 2119 Keywords

This specification uses the following keywords as defined in [RFC 2119](https://www.rfc-editor.org/rfc/rfc2119):

| Keyword | Meaning |
|---------|---------|
| **MUST** | Absolute requirement |
| **MUST NOT** | Absolute prohibition |
| **REQUIRED** | Same as MUST |
| **SHALL** | Same as MUST |
| **SHALL NOT** | Same as MUST NOT |
| **SHOULD** | Recommended, but valid exceptions may exist |
| **SHOULD NOT** | Not recommended, but valid reasons may exist |
| **RECOMMENDED** | Same as SHOULD |
| **MAY** | Optional |
| **OPTIONAL** | Same as MAY |

## Usage

These keywords are written in **UPPERCASE** when used with their RFC 2119 meaning.

When used in lowercase, they have their ordinary English meaning.

## Examples

> Implementations **MUST** preserve the order of postings within a transaction.

> Implementations **SHOULD** provide helpful error messages for syntax errors.

> Implementations **MAY** support additional directive types as extensions.
